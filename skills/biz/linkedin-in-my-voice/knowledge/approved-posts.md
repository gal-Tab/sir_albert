## 2026-07-19 — GTM container migration automated with Claude Code

```
מי שעבד פעם על מיגרצה בין קונטיינרים ב-GTM יודע שזה כואב, לוקחים כל tag, trigger, variable, מעתיקים ידנית, ומקווים/מתפללים שלא שכחת איזה setting קטן שישבור את המדידה בפרודקשן.

אז ניסיתי משהו אחר, בניתי agentic loop עם Claude Code שעובד מול ה-GTM API ומריץ את כל המהלך לבד,

הוא לוקח snapshot של ה-container הקיים, בונה תוכנית migration מסודרת, מבצע אותה tag אחרי tag, ובסוף שולח לי עדכון ב-Slack עם מה שקרה.

החלק המעניין הוא לא שהוא מעביר tags, זה החלק הקל,
החלק הקשה הוא לגרום לו להבין הקשר, מתי setting מסוים חייב להישאר, מתי לעצור ולשאול אותי, ואיך לוודא שה-snapshot באמת תפס את המצב לפני שנגענו במשהו.

בינתיים הכל מחכה לי ב-workspace חדש ב-GTM (לא נותן לו לפבלש לפרודקשן לבד), אבל ההרגשה של לראות migration שלם קורה מול העיניים, ואז נוחת ב-Slack "זה מוכן"...זה עושה לי משהו בלב.
```

**Why it worked:** pain hook everyone in the field feels ("יודע שזה כואב"), flowing comma rhythm, casually Hebraizes some tech terms (מיגרצה/קונטיינרים/עדכון/פבלש) while keeping others in English, names the easy vs hard part honestly, concrete hedge in a parenthetical (new workspace, no auto-publish), warm personal close with an ellipsis instead of a takeaway.
